\section{Experiments}\label{sec:experiments}

\subsection{Experimental Setup}

We train separate instances of the experimental system for four task domains using the protocol described in Section~\ref{sec:method}.
All models use 1000 gradient steps per stage, batch size 32, and support size $N{=}16$.
Evaluation is performed on held-out \emph{OOD families} that were never observed during training.
The primary model applies the deterministic encoder latent $z$ directly to the hypernetwork (no diffusion sampling).
We also evaluate a diffusion add-on that applies DDIM sampling conditioned on the encoder context;
for parameter-reconstruction metrics we report the mean over $K{=}8$ samples.
The \emph{Baseline} is a static MLP with no access to support data, trained jointly as a task-agnostic predictor.
This baseline is learned, but non-adaptive (single policy for all tasks), unlike support-conditioned methods.

\paragraph{Task families.}
\emph{Classification} (8 train / 0 explicit eval families): 2D binary tasks spanning linear, XOR-style nonlinear separability, moons/circles manifolds, sinusoidal boundaries, and checkerboard/ring structure.
\emph{Regression} (6 train / 4 eval): 1D families including piecewise, polynomial, oscillatory, and localized-bump function classes.
\emph{Bandit} (4 train / 2 eval): 2D reward landscapes with linear, quadratic, sinusoidal/radial, and shifted/cubic multimodal structure.
\emph{Control} (8 train / 4 eval): 2D continuous-control dynamics including linear, nonlinear, stiff, drifting, Van der Pol, Duffing, double-integrator, and overdamped systems.

\paragraph{Classification.}
The encoder achieves 83.31\% accuracy on the classification evaluation split, surpassing
the static baseline by 24.8\% relative improvement (baseline: 66.8\%).
The diffusion add-on closely matches the encoder at 83.46\%, consistent with the observation that
classification tasks admit low-variance latent codes---the posterior over classifiers
is near-deterministic given 16 support points.
Per-family breakdown shows encoder accuracy ranges from 67.39\% (imbalanced) to
95.44\% (XOR), with solid generalization across diverse OOD family geometries.

\paragraph{Regression.}
The encoder achieves $R^2 = 0.626$ on the regression evaluation summary, a dramatic 
135.2\% relative improvement over the static baseline ($R^2 = -1.778$), which predicts 
systematic negative variance-explained. The diffusion add-on achieves $R^2 = 0.567$, 
closely tracking the encoder with modest variance. Per-family analysis reveals robust 
generalization: the encoder achieves $R^2 > 0.75$ on sawtooth and cubic-spline families, 
but lower performance ($R^2 \approx 0.55$--$0.59$) on Gaussian-bump and piecewise families,
where localized or discontinuous structure increases ambiguity from sparse support sets.

\paragraph{Bandit.}
Bandit reward-surface reconstruction is the most challenging domain: the encoder 
achieves $R^2 = 0.136$ (still low absolute fit) with substantial improvement 
over baseline ($R^2 = -2.280$). The diffusion add-on achieves $R^2 = 0.110$. 
reflects fundamental difficulty: bandit reward landscapes exhibit high sensitivity to 
policy parameters in 2D action space, and limited support coverage ($N=16$) coupled with 
qualitative regime changes amplifies generalization error. The model correctly identifies 
structural patterns but absolute reconstruction remains challenging.

\paragraph{Control.}
The encoder achieves $R^2 = 0.994$ on control policy parameter reconstruction, 
demonstrating near-perfect reconstruction performance from 16 support trajectories.
The static baseline achieves $R^2 = 0.792$ (25.6\% relative improvement for encoder),
reflecting shared qualitative structure across dynamics families (negative feedback stabilization)
but limited adaptation to fine-grained task geometry. The diffusion add-on achieves $R^2 = 0.989$,
closely tracking the encoder with small variance, validating that the learned latent 
code concentrates near a deterministic optimum.

\paragraph{Cross-domain synthesis.}
The benchmark reveals a clear domain-difficulty hierarchy: control is near-ceiling
($R^2 = 0.994$), classification is strong (83.31\%), regression is moderate but robust
($R^2 = 0.626$), and bandit remains fundamentally hard ($R^2 = 0.136$). The diffusion add-on
tracks encoder performance closely in all domains, with the largest absolute gap in
regression, indicating that most uncertainty arises from task ambiguity rather than
sampling variance in latent space.

\paragraph{Direct-baseline cross-check.}
Using 3-seed support-mode direct baselines, we obtain: classification $0.789\pm0.017$,
bandit $0.142\pm0.033$, control $0.973\pm0.002$, and regression $-0.325\pm0.143$.
Direct baseline is competitive in classification/bandit/control (including a slight edge over
diffusion on bandit in this run), but substantially worse on regression.
This confirms that direct predictions are not uniformly superior; the relative ordering is
task-dependent and should be interpreted per-domain.

\paragraph{OOD sanity check (artifact-backed).}
For runs that include explicit held-out evaluation families in
\texttt{generalization.eval\_summary.overall}, we observe the following OOD metrics:
regression $(0.1770, -0.0372, -5.8345)$ for (encoder, diffusion, baseline),
bandit $(-0.2957, -0.2706, -0.5694)$, and control $(0.9646, 0.9516, 0.7642)$.
The classification run in this set does not include a separate
\texttt{eval\_families} block in its summary artifact.

\paragraph{Control reward semantics.}
For control tasks we report negative quadratic cost as reward; higher (less negative)
is better. A policy \emph{wins} an episode if its cumulative reward exceeds that of the
static baseline on the same episode. We report reward win-rate and mean $\Delta$Reward
(meta-policy minus static baseline) separately from reconstruction $R^2$.
Hence, a negative mean $\Delta$Reward indicates the adaptive policy underperforms the
static learned baseline in cumulative reward under this protocol.

\subsection{Encoding-Mode Ablation}

Beyond using support demonstrations, the model can encode a task via its natural-language
description (\emph{text}), a convex mixture of the two (\emph{hybrid}), or a known
one-hot family index (\emph{oracle}).
Figure~\ref{fig:encoding_modes} and Table~\ref{tab:encoding_modes} compare these modes
on the control task with OOD eval families across three independent seeds.

\begin{figure}[ht]
\centering
\includegraphics[width=0.92\linewidth]{figures/plots/encoding_mode_ablation.png}
\caption{Encoding-mode ablation on control tasks (OOD eval families).
	\emph{Left}: R\textsuperscript{2} for parameter reconstruction---\emph{support} mode
wins because it has direct access to demonstrations. \emph{Right}: reward win-rate
versus the static baseline with 3-seed error bars (mean$\pm$std), including the
direct baseline support-mode point when available. All meta-learned
modes remain below the static baseline in this protocol; \emph{hybrid} has the highest
mean win-rate (26.7\%$\pm$5.1\%), followed by \emph{support} (25.7\%$\pm$12.4\%).
Family descriptions use semantic embeddings from DistilBERT.}
\label{fig:encoding_modes}
\end{figure}

\paragraph{Reconstruction--reward gap.}
A key empirical finding is the \emph{reconstruction--reward gap}: support mode with
$R^2 = 0.969\pm0.006$ (near-perfect parameter recovery) achieves only
$25.7\%\pm12.4\%$ reward win-rate, well below 50\%.
This indicates that accurate parameter-space reconstruction does not directly translate
to better deployed reward: the static baseline happens to produce a reward-competitive
policy for the OOD evaluation dynamics even though it cannot reconstruct those dynamics.
Reward win-rate should therefore be interpreted as a question of task-specific marginal
gain over a strong baseline, not as an absolute measure of adaptation quality.

\paragraph{Text-mode trade-off.}
The \emph{text} encoding trails support mode on parameter reconstruction
($R^2$: $0.812\pm0.051$ vs $0.968\pm0.006$), and also underperforms on
reward win-rate ($4.5\%\pm4.7\%$).
This suggests language descriptions encode useful high-level task intent (linear vs.
non-linear dynamics, stiffness, drift direction) while lacking the fidelity needed for
precise low-level parameter identification.
Text-mode therefore appears insufficient as a standalone modality in this setting.

\paragraph{Hybrid mode.}
The hybrid mode ($\alpha = 0.5$) achieves $R^2 = 0.966\pm0.005$ reconstruction,
close to support mode, while achieving the best mean reward win-rate ($26.7\%\pm5.1\%$).
Combining demonstration-level fidelity with language-level semantic abstraction appears
complementary for reward transfer, suggesting the two modalities carry partially independent
task information.

\paragraph{Oracle collapse.}
The \emph{oracle} mode (one-hot family index) achieves $R^2 = 0.838\pm0.036$ and
reward win-rate $14.6\%\pm24.4\%$, with very high variance across seeds.
This is a counter-intuitive result: knowing the exact family identity is worse for
deployed performance than using noisy language descriptions.
We hypothesise that oracle conditioning causes the model to overfit the mean policy of
each training family, producing weights that interpolate between training-family prototypes
rather than adapt to the specific OOD evaluation instance.
Demonstrations, by contrast, carry instance-level task information that forces adaptation
beyond family-level prototyping.

\paragraph{Static baseline strength and limitation.}
The static baseline achieves mean reward within the range of meta-learned policies,
making it a strong win-rate comparator---a testament to the difficulty of OOD reward transfer
in this evaluation protocol.
However, the baseline consistently exhibits lower parameter-reconstruction $R^2$
and qualitatively fails to capture task-specific geometric structure in held-out dynamics
(e.g., asymmetric restoring forces, stiff gains).
The reconstruction advantage of meta-learned models reflects genuine task-conditional
adaptation; reward advantage remains an open empirical question under longer training.

\subsection{Adaptation to Support Size}

\begin{figure}[ht]
\centering
\includegraphics[width=0.55\linewidth]{figures/plots/support_sweep.png}
\caption{Support-size sweep. Performance improves monotonically with the number of
	support examples, confirming that the model productively integrates additional
	task evidence. The diffusion sampler closely tracks the deterministic encoder.}
\label{fig:support_sweep}
\end{figure}

Figure~\ref{fig:support_sweep} shows performance as a function of the number of
support examples $N \in \{1, 2, 4, 8, 16, 32\}$.
Performance improves monotonically for both encoder and diffusion, confirming that
the model makes productive use of additional evidence.
Diffusion closely tracks the encoder at all support sizes, with slightly larger variance
at $N = 1$ where the latent code is underdetermined.
The gain from $N = 1$ to $N = 4$ is large; gains plateau more slowly beyond $N = 16$,
suggesting the current support size is near the saturation regime for this task class.
